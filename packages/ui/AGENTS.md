# Shared UI (packages/ui)

## Purpose

Reusable UI components for web and (optionally) mobile where feasible.

## Rules

- Components must be accessible by default.
- Keep components focused and composable.
- Avoid app-specific business logic inside UI components.

## Conventions

- Provide sensible defaults and allow overrides via props.
- Include loading/empty/error helpers as shared components.
- Keep styling consistent across the app (single design language).
